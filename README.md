# Онлайн-платформа квеста "Стратегия успешного наставничества"

Этот проект реализует backend-часть образовательной платформы для конкурсного квеста кураторов учебных групп. Основной стек:

- Laravel 10
- MySQL в Docker
- Docker Compose

## Структура проекта

Основные директории:

- `app/` — приложение Laravel
- `database/migrations/` — миграции базы данных
- `database/seeders/` — сидеры для начальных данных
- `public/` — публичная точка входа
- `resources/` — frontend-ресурсы Laravel
- `front/` — Vue-фронтенд web-интерфейса

## Frontend

Каталог `front/` содержит Vue приложение для web-интерфейса. После поднятия backend можно запустить фронтенд из `front/`.

Фронтенд требует Node.js >= 16.8.0.

```bash
cd front
npm install
npm run dev
```

## Быстрый запуск

В этом репозитории используется Docker Compose из корневой директории проекта.

### 1. Скопируйте `.env`

```bash
cd src
cp .env.example .env
```

### 2. Настройте параметры базы данных

В `.env` установите следующие значения:

```env
DB_CONNECTION=mysql
DB_HOST=db
DB_PORT=3306
DB_DATABASE=students
DB_USERNAME=user
DB_PASSWORD=password
```

Если вы хотите использовать SQLite для локальной разработки, оставьте `DB_CONNECTION=sqlite`.

### 3. Запустите Docker

Вернитесь в корень проекта и поднимите контейнеры:

```bash
cd /home/pr26svn/projects/prof
docker compose up -d
```

### 4. Установите зависимости

Если зависимости ещё не установлены внутри контейнера:

```bash
docker compose exec app composer install
```

### 5. Сгенерируйте ключ приложения

```bash
docker compose exec app php artisan key:generate
```

### 6. Выполните миграции и сиды

```bash
docker compose exec app php artisan migrate --seed
```

Если хотите выполнить только миграции без сидов:

```bash
docker compose exec app php artisan migrate
```

## Полезные команды

Рабочая директория: `src`

- `docker compose exec app php artisan migrate`
- `docker compose exec app php artisan migrate --seed`
- `docker compose exec app php artisan db:seed`

# 📖 Руководство пользователя

## 🎯 Обзор платформы

**Платформа квеста "Стратегия успешного наставничества"** — это образовательная система для конкурсного квеста кураторов учебных групп. Платформа позволяет участникам формировать команды, выполнять задания и отслеживать прогресс в режиме реального времени.

## 👥 Роли пользователей

### Участник (User)
Обычный пользователь платформы. Может:
- Регистрироваться и входить в систему
- Просматривать доступные этапы и задания
- Создавать и присоединяться к командам
- Отправлять решения заданий
- Просматривать прогресс своей команды и рейтинг команд

### Администратор (Admin)
Полные права на управление платформой. Может:
- Все возможности участника
- Создавать и редактировать этапы квеста
- Создавать и редактировать задания
- Просматривать все команды и их прогресс
- Оценивать решения участников

## 🚀 Начало работы

### 1. Регистрация и вход

1. Откройте браузер и перейдите на `http://localhost:5173`
2. Нажмите "Вход" или перейдите на страницу `/login`
3. Если у вас нет аккаунта, используйте тестовые учетные данные:
   - **Администратор**: `admin@test.com` / `password`
   - **Участник**: `user1@test.com` / `password`

### 2. Навигация по платформе

После входа вы увидите:
- **Главная страница** — обзор вашего прогресса
- **Команды** — управление командами
- **Этапы** — просмотр доступных заданий
- **Рейтинг** — сравнение прогресса команд

## 👥 Работа с командами

### Создание команды

1. Перейдите в раздел "Команды"
2. Нажмите "Создать команду"
3. Введите название и описание
4. Команда создана! Вы автоматически становитесь её участником

### Присоединение к команде

1. В списке команд найдите интересующую
2. Нажмите "Присоединиться"
3. Вы стали участником команды

### Просмотр прогресса команды

1. Перейдите в раздел "Команды"
2. Выберите свою команду
3. Просмотрите:
   - Список участников
   - Общий прогресс выполнения
   - Детальную статистику

## 📝 Выполнение заданий

### Просмотр доступных заданий

1. Перейдите в раздел "Этапы"
2. Выберите этап квеста
3. Просмотрите список заданий

### Отправка решения

1. Выберите задание
2. Ознакомьтесь с условием
3. Введите ответ в текстовом поле или загрузите файл
4. Нажмите "Отправить"

### Просмотр статуса решения

1. Перейдите в "Мой прогресс"
2. Увидите статус каждого отправленного решения:
   - **Ожидает проверки** — решение отправлено, ждёт оценки
   - **Принято** — решение оценено положительно
   - **Требует доработки** — нужны исправления

## 📊 Рейтинг и статистика

### Просмотр рейтинга команд

1. Перейдите в раздел "Рейтинг"
2. Увидите таблицу команд с процентом выполнения
3. Команды отсортированы по прогрессу

### Личный прогресс

1. На главной странице отображается ваш личный прогресс
2. В разделе "Команды" — прогресс вашей команды
3. В разделе "Этапы" — статус каждого задания

## 🔐 Админ функции

Администраторы платформы имеют расширенные возможности для управления:

### Управление пользователями
- **Просмотр всех пользователей**: `GET /api/admin/users`
- **Создание пользователей**: `POST /api/admin/users`
- **Просмотр пользователя**: `GET /api/admin/users/{id}`
- **Обновление пользователей**: `PUT /api/admin/users/{id}`
- **Удаление пользователей**: `DELETE /api/admin/users/{id}`

### Управление командами
- **Создание команд**: `POST /api/teams` (только для админов)
- **Добавление пользователей в команды**: `POST /api/admin/teams/{team}/add-user`
- **Удаление пользователей из команд**: `DELETE /api/admin/teams/{team}/remove-user/{user}`
- **Управление этапами и заданиями**: полные CRUD операции

### Ролевая модель
- **user**: обычный пользователь (участник квеста)
- **admin**: администратор (управление платформой)
